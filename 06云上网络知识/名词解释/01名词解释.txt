1.  VPC 
    虚拟私有网络(Vritual Private Cloud, VPC)
        是一块在云上自定义的逻辑隔离网络空间，所有云上的服务资源都托管在VPC内，包括"云服务器"、"负载均衡"、
    "云数据库"等等。
        每个私有网络内的服务资源默认内网互通，不同私有网络之间内网默认不通，如果需要和其他的VPC互通，需要
	通过"对等连接"和"云联网"来实现。
	
	    组成部分：
		   (1). 私有网络网段
		            用户在创建VPC时，需要用CIDR(无类别域间路由, 例如10.0.0.0/16, 网络号占16位，剩下的16位都是
				主机号), 作为VPC指定IP地址组。
				
		   (2). 子网
		            一个私有网络至少有一个子网。子网的CIDR必须在VPC的CIDR内。私有网络的所有云资源都必须部署在
				子网内。
				    VPC具有"地域(Region)"属性(例如北京、上海、宿迁)。而子网具有"可用区(Zone)"属性，一个VPC下的
				subnet可以属于该Region下的不同Zone，同一VPC下各个subnet内资源无论是否在同一Zone内，均默认内网
				互通。
			    
		   (3). 路由表
		            每个VPC有一个默认路由表，用户还可以创建自定义路由表。路由表由多条路由策略组成，用于控制VPC
				内子网的出流量走向，每个子网能且只能关联一个路由表，一个路由表可以关联多个子网。可以为不同流量
				走向的子网创建多个路由表。
				    路由策略为：三元组(目的端   下一跳类型   下一跳接口)

2.  DHCP
        动态主机设置协议 (DHCP, Dynamic Host Configuration Protocol) 是一种局域网的网络协议，提供了将配置信息
	传递到TCP/IP网络服务器的标准。
        VPC内的云服务器支持DHCP协议，支持配置的DHCP Options字段包括：DNS地址、Domain Name
        Domain Name是云服务器 hostname 后缀，例如 example.com 		  
	
	
3.  广播和组播
        广播和组播是一对多的通信方式，通过单点到多点的高效数据传送，可以为企业节约网络带宽、降低网络负载
		
		
2.  对等连接( Perring Connection )
        对等连接是一种大带宽、高质量的云上资源互通的服务，对等连接具有多区域、多账户、多种网络异构互通等特点。
	可以帮助客户实现全球同服和两地三中心容灾。
		
		VPC对等连接( VPC Perring Connection )是一种用于办公数据同步的跨VPC网络互联服务，可以使两个VPC之间的
	路由互通，就像它们属于同一网络一样。通过在两端配置路由策略，可实现同地域或跨地域之间，相同或不同用户的
	VPC互联。对等连接不依赖某个独立硬件，因而不存在单点故障或带宽瓶颈的问题。
	
	    对等连接能使VPC两两建立互联，但是这种互通关系不发生传递。
		例如VPC1 和 VPC2建立了对等连接，VPC1 也和 VPC3建立了对等连接，但VPC2 不能通过 VPC1 与 VPC3进行流量互通。
		
		注意：
		   (1). 要使对等连接两端实现真正的通信，必须在发起端和接收端的相关路由表上配置指向对端的路由规则。
		   (2). 对等连接的两端VPC CIDR不可以重叠
		   (3). 对等连接中的任意一方可以随时中断连接。中断后两个VPC间流量会立即中断。
		   (4). 同地域对等连接无带宽上限，跨地域对等连接需要设定带宽上限。
